Overview

WaveStrap is a custom Roblox bootstrapper and runtime controller built around a specific goal: take control of how Roblox is installed, where it is launched from, how launch requests are intercepted, and how the Wave environment stays stable across normal launches, downgraded launches, browser protocol launches, and repair flows.

Based on the structure of the project, WaveStrap is not just a simple launcher window. It is a full launch pipeline made up of setup, configuration, protocol registration, live version tracking, downgrade sourcing, pending URI handling, installation verification, and controlled execution. The app is split into UI windows such as SetupWindow, BootWindow, HomeWindow, and DowngradeWindow, with service classes that manage the system side of the workflow.

What it managesRoblox install discovery, launch routing, downgrade handling, protocol association, runtime preparation, and recovery tools.
Why it existsTo provide a predictable Wave-centered environment instead of depending on Roblox browser bootstrap behavior alone.

Project Architecture

The project layout shows that WaveStrap is organized into separate windows and service layers instead of pushing all behavior into one file. That matters because the app is clearly trying to behave like a real desktop bootstrapper with dedicated responsibilities.

The core structure looks like this: UI windows handle user interaction, service classes handle install and launch logic, API clients fetch version information, and helper classes persist state or register protocol handlers. The updater project exists separately, which suggests the main application and update flow are intentionally isolated.

Setup Experience

The SetupWindow is the first-run configuration experience. It establishes where WaveStrap should store its client data and whether the application should register itself as the handler for Roblox launch requests.

The UI itself matches the rest of the project: dark gradients, rounded chrome, custom title controls, and soft glass panels. More importantly, the code-behind shows what setup actually does.

<Window x:Class="WaveStrap.SetupWindow" Title="WaveStrap Setup" Width="540" Height="420" WindowStyle="None" AllowsTransparency="True">
<TextBox x:Name="PathBox" />
<CheckBox x:Name="ProtocolBox" Content="Set WaveStrap as roblox:// handler" IsChecked="True" />
<Button Content="Save & Continue" Click="Save_Click" />
</Window>
private void Save_Click(object sender, RoutedEventArgs e) {
var cfg = new WaveConfig { InstallPath = PathBox.Text?.Trim() ?? "", Channel = "production", RegisterProtocol = ProtocolBox.IsChecked == true };
if (string.IsNullOrWhiteSpace(cfg.InstallPath)) {
cfg.InstallPath = Path.Combine(Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData), "WaveStrap", "Client");
}
ConfigService.Save(cfg);
if (cfg.RegisterProtocol) ProtocolRegistration.Ensure(robloxHookEnabled: true);
var home = new HomeWindow(); home.Show(); Close();
}

Launch Pipeline

The shared LaunchService is one of the most important parts of WaveStrap. It checks for the live version, searches across possible install roots, can restore Roblox into the expected AppData versions tree, normalizes protocol input, and only then starts the player executable.

public static bool Launch(string? protocol = null) {
try {
string? liveHash = TryGetLiveHash();
var found = FindInstalledPlayer(liveHash);
if (found == null) return false;
var ensured = EnsureRobloxInAppData(found.Value.exe, found.Value.dir, liveHash);
if (ensured == null) return false;
string? uri = NormalizeRobloxPlayerUri(protocol);
return StartRoblox(ensured.Value.exe, ensured.Value.dir, uri);
} catch { return false; }
}
private static bool StartRoblox(string exePath, string workingDir, string? uri) {
var psi = new ProcessStartInfo { FileName = exePath, WorkingDirectory = workingDir, UseShellExecute = true };
if (!string.IsNullOrWhiteSpace(uri)) psi.Arguments = `"${uri}"`;
else psi.Arguments = "--app";
try { return Process.Start(psi) != null; } catch { return false; }
}

Wave Executor Support

WaveStrap helps the Wave ecosystem by making Roblox startup predictable and stable.

  • Starts Roblox directly through a managed process path
  • Supports downgraded clients when compatibility matters
  • Allows launch requests to be intercepted through protocol hooks
  • Prepares multi-instance state before launch
  • Can recover broken or missing installs automatically
