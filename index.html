Overview

WaveStrap is a custom Roblox bootstrapper and runtime controller built around a specific goal: take control of how Roblox is installed, where it is launched from, how launch requests are intercepted, and how the Wave environment stays stable across normal launches, downgraded launches, browser protocol launches, and repair flows.

Based on the structure of the project, WaveStrap is not just a simple launcher window. It is a full launch pipeline made up of setup, configuration, protocol registration, live version tracking, downgrade sourcing, pending URI handling, installation verification, and controlled execution. The app is split into UI windows such as SetupWindow, BootWindow, HomeWindow, and DowngradeWindow, with service classes that manage the system side of the workflow.

What it managesRoblox install discovery, launch routing, downgrade handling, protocol association, runtime preparation, and recovery tools.
Why it existsTo provide a predictable Wave-centered environment instead of depending on Roblox browser bootstrap behavior alone.

Project Architecture

The project layout shows that WaveStrap is organized into separate windows and service layers instead of pushing all behavior into one file. That matters because the app is clearly trying to behave like a real desktop bootstrapper with dedicated responsibilities.

From the code you shared, the core structure looks like this: UI windows handle user interaction, service classes handle install and launch logic, API clients fetch version information, and helper classes persist state or register protocol handlers. The updater project exists separately, which suggests the main application and update flow are intentionally isolated.

Setup Experience

The SetupWindow is the first-run configuration experience. It establishes where WaveStrap should store its client data and whether the application should register itself as the handler for Roblox launch requests. That makes it the first place where WaveStrap turns from a normal desktop app into a launch controller.

The UI itself matches the rest of the project: dark gradients, rounded chrome, custom title controls, and soft glass panels. More importantly, the code-behind shows what setup actually does. It writes a WaveConfig, persists the selected install path, optionally enables protocol registration, and then moves directly into the main launcher window.

"", Channel = "production", RegisterProtocol = ProtocolBox.IsChecked == true }; if (string.IsNullOrWhiteSpace(cfg.InstallPath)) { cfg.InstallPath = Path.Combine( Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData), "WaveStrap", "Client" ); } ConfigService.Save(cfg); if (cfg.RegisterProtocol) ProtocolRegistration.Ensure(robloxHookEnabled: true); var home = new HomeWindow(); home.Show(); Close(); }

Launch Pipeline

The shared LaunchService is one of the most important parts of WaveStrap. It does not assume Roblox is already in the right place. It checks for the live version, searches across possible install roots, can restore Roblox into the expected AppData versions tree, normalizes protocol input, and only then starts the player executable.

That means WaveStrap is acting as a controlled launch pipeline rather than a simple shortcut. The launcher is responsible for making the environment consistent before the process begins.

public static bool Launch(string? protocol = null) { try { string? liveHash = TryGetLiveHash(); var found = FindInstalledPlayer(liveHash); if (found == null) return false; var ensured = EnsureRobloxInAppData(found.Value.exe, found.Value.dir, liveHash); if (ensured == null) return false; string? uri = NormalizeRobloxPlayerUri(protocol); return StartRoblox(ensured.Value.exe, ensured.Value.dir, uri); } catch { return false; } }
private static bool StartRoblox(string exePath, string workingDir, string? uri) { var psi = new ProcessStartInfo { FileName = exePath, WorkingDirectory = workingDir, UseShellExecute = true }; if (!string.IsNullOrWhiteSpace(uri)) psi.Arguments = $"\"{uri}\""; else psi.Arguments = "--app"; try { return Process.Start(psi) != null; } catch { return false; } }

Wave Executor Support

WaveStrap clearly helps the Wave ecosystem by making Roblox startup predictable. Since Roblox updates can change launch behavior, install paths, and version availability, an executor or related tooling benefits from a launcher that stabilizes those details.

WaveStrap helps by handling version-aware launch decisions, routing protocol launches through itself, preparing multi-instance conditions, and allowing downgraded versions to be used when needed. Instead of the rest of the Wave ecosystem depending on Roblox’s default bootstrap path, it can depend on WaveStrap’s controlled startup behavior.

  • Starts Roblox directly through a managed process path
  • Supports downgraded clients when compatibility matters
  • Allows launch requests to be intercepted through protocol hooks
  • Prepares multi-instance state before launch
  • Can recover broken or missing installs automatically

Version Tracking and Downgrade

WaveStrap is version-aware in two different directions. It can ask Roblox for the current live WindowsPlayer version, and it can also query external APIs for past versions. This is what makes downgrade support possible.

The LiveVersionTracker fetches the active LIVE version from Roblox’s client settings endpoint. The RddClient and WeaoClient talk to Wave-related services that expose past version information. Together, they allow WaveStrap to identify both the present version state and a target version for downgrade flows.

internal static class LiveVersionTracker { private static readonly HttpClient _http = new HttpClient { Timeout = TimeSpan.FromSeconds(15) }; public static async Task<string> GetCurrentLiveAsync() { try { var json = await _http.GetStringAsync( "https://clientsettingscdn.roblox.com/v2/client-version/WindowsPlayer/channel/LIVE" ); using var doc = JsonDocument.Parse(json); return doc.RootElement.GetProperty("clientVersionUpload").GetString() ?? ""; } catch { return ""; } } }
internal sealed class RddClient { private static readonly string[] ApiDomains = { "https://weao.xyz/api/status/exploits/wave" }; public async Task<string> GetWindowsPastVersionAsync() { var json = await _http.GetStringAsync(ApiDomains[0]); using var doc = JsonDocument.Parse(json); if (!doc.RootElement.TryGetProperty("rbxversion", out var rbx)) throw new Exception("API did not return rbxversion."); return rbx.GetString()!; } public string BuildRddDownloadUrl(string version) { return $"https://rdd.weao.gg/?channel=LIVE&binaryType=WindowsPlayer&version={version}"; } }
internal static class WeaoClient { public static async Task<string> GetPreviousLiveWindowsPlayerAsync() { const string url = "https://weao.xyz/api/versions/past"; using var req = new HttpRequestMessage(HttpMethod.Get, url); req.Headers.UserAgent.ParseAdd("WaveStrap/1.0"); using var res = await _http.SendAsync(req); res.EnsureSuccessStatusCode(); var json = await res.Content.ReadAsStringAsync(); var match = Regex.Match(json, @"version-[a-f0-9]{16,}", RegexOptions.IgnoreCase); if (!match.Success) throw new Exception("Could not extract Roblox version hash from weao response."); return match.Value; } }

Protocol Registration

Protocol registration is one of the features that makes WaveStrap behave like a true bootstrapper. Instead of only being launched manually, it can register itself as the handler for both a custom wave scheme and the roblox-player scheme.

That means browser play actions or direct protocol links can be redirected into WaveStrap first. Once intercepted, WaveStrap decides how to handle the launch, which allows the rest of the environment to stay consistent.

public static void Ensure(bool robloxHookEnabled) { Register(WaveScheme); if (robloxHookEnabled) { if (!IsHookedToThisExe(RobloxScheme)) Register(RobloxScheme); } else { UnregisterHKCU(RobloxScheme); } }
private static void Register(string scheme) { var exe = GetExePath(); using var key = Registry.CurrentUser.CreateSubKey($@"Software\Classes\{scheme}"); key.SetValue("", $"URL:{scheme} Protocol"); key.SetValue("URL Protocol", ""); using var cmd = key.CreateSubKey(@"shell\open\command"); cmd.SetValue("", $"\"{exe}\" \"%1\""); }

Pending Launch Handling

The PendingLaunch helper shows that WaveStrap is designed to preserve launch state when needed. Incoming URIs can be written into a file under LocalAppData, then read back and cleared later. This is useful when a launch request needs to survive a boot or setup transition before it is actually processed.

That might look like a small helper file, but architecturally it is important. It shows that WaveStrap supports launch orchestration, not just immediate execution.

public static class PendingLaunch { private static string DataDir => Path.Combine( Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData), "WaveStrap", "Data" ); public static string PendingPath => Path.Combine(DataDir, "pending_uri.txt"); public static void Write(string uri) { Directory.CreateDirectory(DataDir); File.WriteAllText(PendingPath, uri); } public static string? ReadAndClear() { try { if (!File.Exists(PendingPath)) return null; var uri = File.ReadAllText(PendingPath).Trim(); File.Delete(PendingPath); return string.IsNullOrWhiteSpace(uri) ? null : uri; } catch { return null; } } }

Install and Repair Services

InstallService gives WaveStrap real recovery capability. If the correct live version is missing or invalid, WaveStrap downloads the official Roblox installer, launches it, waits for a valid version folder to appear, and then cleans up redundant installer files. It can also force a clean install by removing version folders first.

This means WaveStrap is not passive. It does not just fail when Roblox is broken. It actively repairs the install chain and brings the client back into a usable state.

public static async Task<string> EnsureRobloxInstalledAndGetLatestVersionDirAsync(Action<string>? log = null) { log ??= _ => { }; string liveHash = await LiveVersionTracker.GetCurrentLiveAsync(); string targetDir = Path.Combine(LocalRobloxVersions, liveHash); if (!Directory.Exists(targetDir) || !IsValidRobloxVersionFolder(new DirectoryInfo(targetDir))) { log("Correct version not found. Launching official installer..."); var installerPath = Path.Combine(Path.GetTempPath(), "RobloxPlayerInstaller.exe"); await DownloadFileAsync("https://www.roblox.com/download/client?os=win", installerPath, log); var process = Process.Start(new ProcessStartInfo { FileName = installerPath, UseShellExecute = true }); if (process != null) await process.WaitForExitAsync(); CleanupAllInstallers(); return (await WaitForLatestValidVersionAsync(TimeSpan.FromMinutes(8), log)).FullName; } return targetDir; }
public static async Task ForceCleanInstallAsync() { foreach (var p in Process.GetProcessesByName("RobloxPlayerBeta")) try { p.Kill(); } catch { } if (Directory.Exists(LocalRobloxVersions)) { foreach (var dir in Directory.GetDirectories(LocalRobloxVersions, "version-*")) try { Directory.Delete(dir, true); } catch { } } await EnsureRobloxInstalledAndGetLatestVersionDirAsync(); }
